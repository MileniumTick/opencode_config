---
description: Generate a conventional commit from staged/unstaged changes
---

Analyze the current git changes and create a conventional commit message. Follow these steps:

1. Run `git status` to see the current state.
2. Run `git diff` (unstaged) and `git diff --cached` (staged) to understand what changed.
3. If nothing is staged, run `git add -A` to stage all changes — but CONFIRM with the user first.
4. Analyze the diff to determine:
   - **type**: feat | fix | refactor | chore | docs | test | style | perf | ci | build
   - **scope**: the module or area affected (optional, keep it short)
   - **subject**: imperative mood, lowercase, no period, max 72 chars
   - **breaking change**: if any, add `BREAKING CHANGE:` to the commit body
5. Show the proposed commit message and wait for user approval:

```
type(scope): subject

[optional body — explain WHY, not WHAT]

[optional footer — e.g., Closes #123]
```

6. Once approved, execute:
```
git commit -m "type(scope): subject"
```

**Rules:**
- NEVER add "Co-Authored-By" or any AI attribution
- NEVER include "Generated by AI" or similar in the message
- Keep the subject line under 72 characters
- Use English for the commit message regardless of user language
- If there are multiple logical changes, suggest splitting into separate commits
