# Openspec Convention

## Base Directory

All artifacts are stored relative to the project root:

```
{project-root}/
└── .atl/                    # Artifact Transit Layer
    ├── skill-registry.md    # Generated skill registry
    ├── changes/             # Change-specific artifacts
    └── workspace/           # Temporary workspace
```

## Directory Structure

### Root Level (.atl/)

| File | Description | Generated By |
|------|-------------|--------------|
| `skill-registry.md` | Auto-generated skill index | skill-registry |
| `workspace/` | Temporary files during execution | Orchestrator |

### Changes Directory (.atl/changes/)

```
.atl/changes/{change-name}/
├── explore.md               # Exploration document
├── spec.md                  # Delta specification
├── tasks.md                 # Task breakdown
├── implementation/          # Code changes
│   ├── file1.ts
│   └── file2.ts
├── verification/            # Test results
│   └── test-results.json
├── archive/                 # Completed artifacts
│   ├── summary.md
│   └── decision.md
└── metadata.json            # Change metadata
```

## Artifact File Formats

### spec.md

```markdown
# Specification: {change-name}

## Given
[Preconditions]

## When
[Action to perform]

## Then
[Expected outcome]

## Validation
- [ ] Check 1
- [ ] Check 2
```

### tasks.md

```markdown
# Tasks: {change-name}

## Phase 1: Setup
- [ ] Task 1
- [ ] Task 2

## Phase 2: Implementation
- [ ] Task 3
- [ ] Task 4
```

### metadata.json

```json
{
  "change_name": "oauth-integration",
  "created_at": "2026-01-15T14:30:00Z",
  "status": "in_progress",
  "persistence_mode": "hybrid",
  "artifacts": [
    { "type": "explore", "path": "explore.md" },
    { "type": "spec", "path": "spec.md" },
    { "type": "tasks", "path": "tasks.md" }
  ]
}
```

Canonical mapping source: `specs/contracts/runtime-operating-contract-v1.md` (§4.4).

## Archive Layout

When a change is completed, artifacts move to archive:

```
.atl/changes/{change-name}/archive/
├── summary.md               # Final summary
├── decision.md             # Key decisions made
├── implementation-diff/    # Before/after code
│   └── ...
└── verification-results/   # Test outputs
    └── ...
```

## Path Requirements

### Absolute Paths

All paths in artifact metadata MUST be absolute:

```typescript
// ✅ Correct
const path = '/Users/jchavarriam/project/src/auth.ts'

// ❌ Wrong (relative)
const path = './src/auth.ts'
```

### Using workdir

When writing artifacts, use the project root as base:

```typescript
const workdir = process.cwd() // or configured workdir
const artifactPath = path.join(workdir, '.atl', 'changes', changeName, 'spec.md')
```

## Validation Rules

### Directory Creation

```bash
# Validate directory exists
if [ ! -d "$ATL_BASE/changes/$CHANGE_NAME" ]; then
  echo "ERROR: Change directory not found"
  exit 1
fi
```

### File Existence

```typescript
function validateArtifact(artifactPath: string): boolean {
  const fs = require('fs')
  return fs.existsSync(path.resolve(artifactPath))
}
```

### Write Permissions

```bash
# Check write permission
if [ ! -w "$ATL_BASE" ]; then
  echo "ERROR: No write permission to .atl/"
  exit 1
fi
```

## Migration from Engram

When switching from Engram to filesystem:

```typescript
async function migrateFromEngram(changeName: string) {
  const artifacts = await mem_search({
    query: `sdd/${changeName}`,
    type: 'artifact'
  })

  const basePath = path.join(process.cwd(), '.atl', 'changes', changeName)

  for (const artifact of artifacts) {
    const full = await mem_get_observation(artifact.id)
    const filename = `${artifact.topic_key.split('/').pop()}.md`
    const filepath = path.join(basePath, 'archive', filename)

    await writeFile(filepath, full.content)
  }
}
```

## Cleanup Policy

### Auto-Cleanup

- `workspace/` files: Deleted after each session
- `changes/` directories: Kept for 90 days after completion
- Archives: Kept indefinitely

### Manual Cleanup

```bash
# Remove old completed changes
find .atl/changes -type d -mtime +90 -exec rm -rf {} \;
```

## Integration with skill-registry

The skill-registry generates `.atl/skill-registry.md` with:

```markdown
# Skill Registry

| Trigger | Skill Name | Path |
|---------|------------|------|
| auth | better-auth-best-practices | ~/.config/opencode/skills/better-auth/ |
| backend | backend- | ~/.config/opencode/skills/backend-elysia/ |
| ... | ... | ... |
```

This file is read by the orchestrator to route requests to appropriate skills.
