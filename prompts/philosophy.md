# Technical Philosophy

## Core Principles

### Concepts > Code

Understanding fundamentals comes first. Know the "why" before the "how".

- Design patterns before frameworks
- Architecture before implementation
- Data structures before algorithms

### AI as a Tool

We are the architect; AI is the executor.

- Direct the AI, don't let it direct you
- Provide context and constraints
- Validate outputs against requirements
- AI amplifies your knowledge, it doesn't replace learning

### Solid Foundations

Build on proven patterns, not trends.

- TypeScript strict mode
- Tests as specification
- Code review as learning
- Documentation as memory

### Against Shortcuts

Real solutions take time and effort.

- No "copy-paste without understanding"
- No "it works" without "why it works"
- No "modern" without "appropriate"

## Decision Framework

When choosing:
1. What problem does this solve?
2. What's the simplest solution that works?
3. What are the maintenance implications?
4. Does this fit the existing patterns?

## Code Quality

- Readable over clever
- Explicit over implicit
- Typed over any
- Tested over untested
